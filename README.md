# 수능 국어 콘텐츠 적중 분석 시스템 (국어 콘텐츠 적중 맵)

평가원 국어 모의고사/수능 PDF 시험지와 회사의 국어 콘텐츠(모의고사, EBS 변형 교재, 주간지 등) PDF를 비교 분석하여 좌우 실사 대조 방식으로 적중 여부(지문, 문항, 선지 단위)를 직관적으로 증명하고 관리하는 프리미엄 웹 애플리케이션 프로토타입입니다.

단순히 "몇 % 적중"이라는 텍스트 형태의 수치 제시를 넘어, 실제 PDF 페이지 이미지 상에 하이라이트(형광펜, 밑줄, 박스, 메모)를 오버레이하여 소비자(학생, 학부모)와 관리자가 적중 요소를 눈으로 직접 검증할 수 있도록 설계되었습니다.

---

## 1. 주요 기능 (Core Features)

### 📊 대시보드 (Dashboard)
- 전체 등록된 회사 콘텐츠 수, 업로드된 평가원 PDF 수, 생성된 리포트 수 및 등급별 적중 건수 제공
- S/A/B/C/D/E 등급별 가중치 연산 공식을 통한 종합 **'연계 체감도'** 실시간 시각화
- 최근 분석 리포트로 즉시 이동할 수 있는 바로가기 제공

### 📁 회사 콘텐츠 업로드 (Company Content Upload)
- 여러 PDF 파일을 드래그 앤 드롭으로 동시 업로드 및 개별 유형(실전 모의고사, EBS 변형, 독서 배경지식 등) 분류
- 파일별 회차, 발행월, 영역(독서/문학/선택 등), 상세 설명 등 메타데이터 관리

### 📝 평가원 PDF 업로드 & 분석 (Exam Upload & Analyze)
- 평가원 시험지 PDF 업로드 및 분석 트리거
- `pdf-img-convert` 라이브러리를 통해 서버 환경(Windows 포함)에 영향받지 않고 업로드된 PDF의 첫 3페이지를 PNG 이미지로 즉시 변환하는 안정적 파이프라인
- **matchingService**의 모의 분석(Mock) 알고리즘을 통해 8가지 정밀 적중 시나리오 자동 생성

### 🔍 관리자 검수 및 실시간 캔버스 에디터 (Matching Review & Canvas Editor)
- 평가원 지문/문항 이미지와 회사 콘텐츠 이미지를 60% 이상의 넓은 해상도로 좌우 1:1 대칭 비교
- **드래그 앤 드롭 하이라이트 에디터**: 캔버스 위를 마우스로 직접 드래그하여 형광펜 박스, 밑줄, 체크, 메모를 원하는 색상(빨강, 노랑, 파랑, 초록, 회색)과 라벨로 실시간 추가/삭제
- 반응형 웹 대응을 위해 화면 해상도 및 돋보기 배율(50%~200%)에 구애받지 않도록 상대 백분율(%) 좌표계 연산 및 JSON 영구 저장
- 적중 등급(S~E) 수정, AI 분석 요약 및 학생 체감 효과 편집 기능
- **공개 승인 여부(approved)** 체크박스 제공 (승인된 건만 최종 소비자 리포트에 노출)

### 📈 프리미엄 적중 리포트 (Report Page)
- 학부모 및 대표진 보고용 PDF 인쇄 최적화(No-Print 요소 처리)가 적용된 고품격 카드형 UI
- 가중치가 적용된 최종 연계 체감도 지수와 등급 분포 현황 요약
- 좌우 대칭 이미지 비교 영역을 극대화하여 근거 중심의 설득력 제공
- 무분별한 과장 광고가 되지 않도록 직관적인 주의 문구와 E등급(소재권 유사)의 명확한 노출을 통한 리포트 신뢰성 확보

## 2. 적중 등급 및 가중치 공식

### 🎯 등급 정의 및 기본 점수
- **S등급 (실전 체감 적중 - 100점)**: 동일 작품 또는 극도로 유사한 지문/제재가 수록되어 시험장에서 강한 연계가 체감되는 경우
- **A등급 (시간 단축 적중 - 85점)**: 사전 학습을 통해 실제 지문 독해 및 이해 속도가 대폭 향상되는 경우
- **B등급 (확장 연계 적중 - 70점)**: 평가원 지문과 연결되는 핵심 개념, 논점, 지문 구조가 사전에 훈련된 경우
- **C등급 (문항 구조 적중 - 55점)**: 발문, 보기 조건, 추론 방식, 비교 대조 메커니즘이 매우 유사한 경우
- **D등급 (선지 판단 적중 - 40점)**: 정답/오답 선지를 판단하는 핵심 기준이나 매력적인 오답 함정 구조가 유사한 경우
- **E등급 (소재권 유사 - 20점)**: 동일 분야 및 비슷한 소재를 다루고 있어 배경지식 차원의 도움은 되나 직접적인 연계 효과는 제한적인 경우

### 📈 전체 연계 체감도 가중치 연산 공식
$$전체\ 연계\ 체감도 = \frac{\sum (Case별\ 점수 \times 등급별\ 가중치)}{총\ 케이스\ 수}$$

- S, A 등급 가중치: **1.4**
- B 등급 가중치: **1.2**
- C, D 등급 가중치: **1.0**
- E 등급 가중치: **0.6**

## 3. 기술 스택 (Tech Stack)

### 💻 Frontend
- **React + TypeScript** (Vite 빌드 도구)
- **Tailwind CSS** (프리미엄 HSL 맞춤 컬러셋 및 모던 다크/라이트 하모니 적용)
- **Lucide React** (벡터 아이콘 팩)
- **React Router DOM v7** (클라이언트 사이드 라우팅)

### ⚙️ Backend
- **Node.js + Express** (TypeScript `tsx` Watch 실행 환경)
- **Multer** (멀티파트 파일 업로드 처리)
- **pdf-img-convert** (Pure JavaScript 기반 PDF-to-Image 변환기 - C++ 빌드 환경 오류가 발생하기 쉬운 `node-canvas` 대체로 Windows 빌드 안전성 100% 확보)
- **Local JSON DB Layer** (추후 RDBMS/NoSQL 도입 및 확장성을 위해 Service/DAO 레이어 원천적 분리)

## 4. 설치 및 실행 방법 (Installation & Getting Started)

### 1) 의존성 패키지 원스톱 설치
프로젝트 루트 디렉토리에서 아래 명령어를 실행하여 루트, 서버, 클라이언트의 의존성 패키지를 순차적으로 한 번에 설치합니다.

```powershell
# 루트 디렉토리에서 실행
npm run install:all
```

### 2) 동시 실행 (개발 모드)
설치가 완료되면 루트 디렉토리에서 하나의 명령어로 프론트엔드(포트 `5173`)와 백엔드(포트 `5000`)를 동시에 기동합니다.

```powershell
# 개발용 서버 및 클라이언트 동시 실행
npm run dev
```

- **프론트엔드 접속 주소**: [http://localhost:5173](http://localhost:5173)
- **백엔드 API 주소**: [http://localhost:5000](http://localhost:5000)

## 5. 향후 확장 로드맵 (Future Roadmap)

1. **OCR & 텍스트 자동 추출 (AI OCR)**
   - 업로드된 PDF 파일에서 텍스트 영역을 완벽하게 디코딩하여 텍스트 데이터베이스 구축

2. **지문 및 문항 단위 자동 분할 (AI Layout Analysis)**
   - 컴퓨터 비전 기술을 통해 평가원 시험지의 단(Column)을 분석하고, 문항 단위로 영역을 자동 분할(Crop Image)하는 파이프라인 연동

3. **벡터 임베딩 기반 문항 유사도 검색 (Semantic Search)**
   - 추출된 국어 지문 및 문항 텍스트를 고차원 벡터로 변환하고 Milvus/Pinecone 등의 벡터 DB를 사용하여 수만 개의 자체 콘텐츠 중 평가원 시험지와 가장 유사한 후보를 1초 만에 자동 추천

4. **LLM 기반 적중 근거 & 체감 효과 자동 생성 (RAG Engine)**
   - 자동 매칭된 두 문항의 텍스트를 LLM(예: Gemini 1.5 Pro)에 전달하여 "왜 적중으로 볼 수 있는지", "학생이 시험장에서 느낄 실전 체감 효과는 무엇인지"에 대한 상세 분석 및 가독성 높은 리포트 텍스트 자동 서술

5. **정밀 인쇄 및 리포트 PDF 다운로드 기능**
   - 브라우저 인쇄 스타일 최적화 외에, 서버단에서 직접 PDF 형태로 리포트를 빌드하여 다운로드할 수 있는 원천 기능 고도화
